  • 新接触的框架

    • kissy 这是一个跨终端开发的框架，在外网同样有关于kissy的技术博客。感觉他就是一个类似与jQuery的JavaScript框架。不过相比之下他在跨终端方面更加的流畅，而且有高度的模块化设计（虽然对于模块化我还处在一知半解的状态吧。）看kissy的过程中遇到的第一个问题是用终端安装git以及kpm来下载安装kissy。由于在校期间一直使用Github的图形化界面来同步代码，所以现在直接使用终端的确会有一些难以理解和记忆的地方。好在后来花了半天的时间大致熟悉了git的基本命令，当然关于git这部分后面再说。终于安装好了kissy之后准备正式进入到开发步骤的时候发现我又不会用node作为服务器来启动kissy的服务。好吧，这依旧是在校期间偷懒没有学习node给自己埋下的坑。请教师姐之后师姐帮我装好了def(Development Environment for Frontend)，def是一个从用户机器到各种线上服务一整套前端开发工具的通用运行环境（这句话是粘贴的～）。def的学习放在后面再说，基于def我又安装了xcake（一个前端应用模型）之后终于可以初始化一个项目框架，开始对kissy各个api进行了解并且尝试使用。然后发现kissy的语法在很大程度上和jQuery还是很相似的，鉴于这一点我用了两天的时间熟悉了kissy的使用方法。就在介个时候，老大给我一个关于双十二的简单任务，于是。。。。转战 -->

    • kimi 这个在外网上并没有搜到，可能是林志颖的儿子太红了的缘故。。。。因为是一个移动端的活动页面，所以要使用kimi进行开发，不过所有前期的配置步骤和kissy还是很相似的，所以在申请到权限的当天就开始学习了kimi的语法，其实和kissy也有很大的相似之处。然后勒，我又用了两天的时间完成了三个模块的小任务。而在这个移动端项目的编写过程中又接触到了 -->

    • iScroll 这是一个高性能支持多平台的JavaScript滚动插件，主要用于移动端一些关于滚动事件的封装，这个插件的使用方法非常简单，引用他的js文件然后实例化IScroll对象，然后传入相应的配置参数就可以了。在移动端的滚动方面，他是一个灰常好用的工具，嗯，卧室这样认为的。

    • Midway 介个勒，是一个前后端分离的中间件，今天大致看了一下只了解了运作方式，还没有开始实践（因为SDK安装失败。。。）。所以也就没有什么可以说得了。

  • 开发工具

    • node 哎～说到node就心酸，当年在学校的时候满满的都是时间，每每想到该要看看node的时候就再给自己找一个借口来推迟node的学习计划，以至于我到来工作对node的了解都只是皮毛，来工作之后发现整个前端开发的过程都离不开node，首先所有的框架类开发都是基于node服务器的，其次产品和前端合作数据录入的TMS也是使用node来取数据的。所以我这个对node一无所知的渣渣再次遇到一个问题就是我可以按照说明文档给的demo进行开发但是页面都开发完了我不知道为啥要这么写。。。。这真的好可怕！！所以我就赶快去了解了node的皮毛。但是现在对node的了解真的只是皮毛而已，所以下面还要多花一点时间在node上。

    • git 嘿嘿嘿，我买了那本《Git版本控制管理》。下单的时候看着那本书挺薄的呀，到货之后看到哪本书的厚度我又一次被吓到了。。。现在对git的了解也仅限于创建一个仓库，clone一个仓库，提交一个版本，创建分支，更新本地仓库这些简单的命令，连配置用户名和邮箱都是找师兄帮忙的。。。。哎～少壮不努力，老大徒伤悲呀！！

    • 还有一些开发用的工具也只是知道怎么使用而已，不知道具体到底是个啥，所以就在这写吧了～

  • 小知识

    • rem rem这个单位之前在翻译译文的时候就有遇到过，但是当时的译文重点不在这上面所以也没有去深入了解这个移动端单位的使用方法，在这次移动端界面的开发过程中因为使用到了rem所以对它有了一个全面的理解，那天写一篇关于rem的博客吧～

    • new Date() 在建立这个日记站点的过程中发现一个问题就是在其他浏览器都会正常显示，但是在Safari上面切换月份就回出现年份月份日期都不显示的bug，搜索发现原来Safari对Date()传入的参数有要求。之前我传入的日期是类似于2016 11 06 这个写法Chrome可以识别但Safari不可以，所以改成2016/11/06就可以啦～
